# 2P_opto_tools
Code related to optogenetics and holography experiments.

## Experiment workflow

### Prepare in advance

1. A pattern file which contains multiple stimulation patterns. Each pattern is a combination of cells, defined in `x,y,z` coordinates, that will be stimulated for some period of time and some frequency and power. For example, one pattern could be: cell1 `(x20,y34,z100)` and cell2 `(x40,y80,z400)` stimulated for 1 second at 10 Hz at 20% power. Each pattern will become one ScanImage **stimulus group**.
2. A sequence file which contains multiple sequences. Each sequence is a list of patterns and the time relative to trial onset when each should be triggered. Patterns should not overlap in time, and the pattern file generation code guarantees this.

### During experiment

1. bvGUI requests ScanImage pregeneration of all patterns that will be used in the experiment. This includes generation of all required phase masks and upload of these to the SLM so that they can be triggered with short latency.
2. After generation of all phase masks associated with all stimulus groups, ScanImage confirms completion via a UDP message to bvGUI and the experiment is ready to continue.
3. On each trial, bvGUI requests ScanImage to prepare to execute a sequence of stimulus groups by requesting a sequence from the sequence file. This involves:
4. First updating the current **Stimulus Group Sequence**.
5. Second preparing to play a waveform, generated by the ScanImage DAQ, which can be externally triggered at trial onset and contains digital triggers at the times when the stimulus group should be changed relative to trial start. This waveform will be fed back as a trigger source in the photostimulation controls. The execution of this waveform will be triggered by a digital trial onset signal from BonVision.
6. ScanImage then confirms via UDP that it is ready for the trial to start.
7. At trial onset, BonVision provides a trial start digital signal that triggers the ScanImage stimulus group sequence timing waveform and precisely timed phase mask updates and modulations of laser power as defined in each stimulus group.

## Pattern Builder GUI

PyQt editor for one YAML schema file containing both stimulation patterns and sequences.

### Install

```bash
pip install -e .
```

### Run

Without installing:

```bash
python run_pattern_builder_gui.py
```

If you prefer a console entry point:

```bash
opto-schema-gui
```

### File format

The app writes one YAML file:

- `schema.yaml`

It contains both `patterns` and `sequences`.

Each sequence step stores the pattern name, `start_s`, and computed `end_s` so downstream code can recover transition times without recomputing them.

### Save location

The app reads `config.ini` in the repo root and uses `paths.save_root` as the base output directory.

Default save path:

- `<save_root>/<animal_id>/<project_name>/schema.yaml`
