# 2P_opto_tools
code related to optogenetics/holography experiments
Procedure for running experiment

**Prepare in advance:**
1) A pattern file which contains multiple stimulation patterns. Each pattern is a combination of cells (defined in x,y,z coordinates) that will be stimulated for some period of time and some frequency and power. For example one pattern could be: cell1 (x20,y34,z100) and cell2 (x40,y80,z400) stimulated for 1 second at 10Hz at 20% power. Each pattern will become one Scanimage **stimulus group**.
2) A sequence file which contains multiple sequences. Each sequence is a list of patterns and the time relative to trial onset when each should be triggered. Patterns should not be overlapping in time and pattern file generation code guarentees this.

**During experiment:**
1) bvGUI requests to Scanimage pregeneration of all patterns that will be used in the experiment.  This will include generation of all required phase masks and upload of these to the SLM such that they can be triggered at short latency.
2) After generation of all phase masks associated with all stimulus groups, Scanimage confirms completion via a UDP msg to bvGUI and experiment is ready to continue.
3) On each trial, bvGUI requests Scanimage to prepare to execute a sequence of the stimulus groups by requesting a sequence from the sequence file. This involves:

5) **Second** preparing to play a wavefrom (generated by the scanimage DAQ) which can be externally triggered at trial onset, which contains digital triggers at the times when the stimulus group should be changed relative to trial start. This waveform will be fed back as a trigger source in the photostimulation controls. The execution of this waveform will be triggered by a digital trial onset signal from bonvision.
